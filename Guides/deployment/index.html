    üöÄ Deployment Guide¶

    Overview¶

    This guide covers deploying your Telegram bot to production using Docker and Railway. The setup includes automatic scaling, monitoring, and secure environment management.

    üê≥ Docker Configuration¶

    Dockerfile¶

    # Use Python 3.11 slim image
FROM python:3.11-slim

# Set working directory
WORKDIR /app

# Install system dependencies
RUN apt-get update && apt-get install -y \
    gcc \
    libpq-dev \
    && rm -rf /var/lib/apt/lists/*

# Copy requirements first for better caching
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy application code
COPY src/ ./src/
COPY scripts/ ./scripts/

# Create non-root user for security
RUN useradd --create-home --shell /bin/bash app
USER app

# Expose port
EXPOSE 8000

# Health check
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
    CMD curl -f http://localhost:8000/health || exit 1

# Start the application
CMD ["gunicorn", "--bind", "0.0.0.0:8000", "--workers", "2", "--timeout", "30", "src.webhook_server:app"]

    Step 1: Prepare Repository¶

    Step 2: Railway Setup¶

    1. Connect Repository: 

      # Install Railway CLI
npm install -g @railway/cli

# Login and connect
railway login
railway link

    2. Configure Environment Variables: 

      # Set required environment variables
railway variables set BOT_TOKEN="your_bot_token"
railway variables set STRIPE_API_KEY="sk_live_..."
railway variables set STRIPE_WEBHOOK_SECRET="whsec_..."
railway variables set DATABASE_URL="postgresql://..."
railway variables set ADMIN_GROUP_ID="-1001234567890"

    Step 3: Database Setup¶

    # Add PostgreSQL service
railway add postgresql

# Run database setup script
railway run python scripts/setup_db.py

    Step 4: Deploy¶

    # Deploy to production
railway up

    üîß Environment Configuration¶

    Required Environment Variables¶

    Variable Description Example
    BOT_TOKEN Telegram bot token from BotFather 1234567890:AAE...
    DATABASE_URL PostgreSQL connection string postgresql://user:pass@host:5432/db
    STRIPE_API_KEY Stripe secret key sk_live_...
    STRIPE_WEBHOOK_SECRET Stripe webhook signing secret whsec_...
    ADMIN_GROUP_ID Telegram admin group ID -1001234567890
    ENVIRONMENT Deployment environment production

    üîê SSL/TLS Configuration¶

    Automatic HTTPS with Railway¶

    Railway automatically provides SSL certificates for your domain:

    # webhook_server.py
from flask import Flask

app = Flask(__name__)

@app.route('/health')
def health_check():
    return {"status": "healthy", "timestamp": datetime.utcnow().isoformat()}

if __name__ == '__main__':
    # Railway handles SSL termination
    app.run(host='0.0.0.0', port=int(os.environ.get('PORT', 8000)))

    Custom Domain Setup¶

    1. Add Custom Domain in Railway Dashboard

    2. Update DNS Records: 

      Type: CNAME
Name: bot (or your subdomain)
Value: your-app.railway.app

    3. Update Webhook URL: 

      webhook_url = "https://bot.yourdomain.com/telegram-webhook"

    Health Check Endpoint¶

    from flask import Flask, jsonify
from datetime import datetime
import psycopg2

@app.route('/health')
def health_check():
    try:
        # Check database connection
        conn = get_db_connection()
        cursor = conn.cursor()
        cursor.execute("SELECT 1")
        cursor.close()
        conn.close()

        return jsonify({
            "status": "healthy",
            "timestamp": datetime.utcnow().isoformat(),
            "database": "connected"
        }), 200
    except Exception as e:
        return jsonify({
            "status": "unhealthy",
            "error": str(e),
            "timestamp": datetime.utcnow().isoformat()
        }), 500

    Logging Configuration¶

    import logging
import sys

def setup_logging():
    logging.basicConfig(
        level=logging.INFO,
        format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
        handlers=[
            logging.StreamHandler(sys.stdout),
            logging.FileHandler('bot.log')
        ]
    )

# Usage
setup_logging()
logger = logging.getLogger(__name__)

    Common Issues¶

    Issue Symptoms Solution
    Bot not responding Messages not received Check webhook URL and SSL certificate
    Database connection failed 500 errors Verify DATABASE_URL and network connectivity
    Stripe webhooks failing Payments not processed Check webhook secret and endpoint URL
    High memory usage App restarts frequently Implement connection pooling and caching

    Performance Optimization¶

    # Connection pooling for database
from psycopg2 import pool

class DatabasePool:
    def __init__(self):
        self.connection_pool = psycopg2.pool.SimpleConnectionPool(
            1, 20,  # min and max connections
            host=DB_HOST,
            database=DB_NAME,
            user=DB_USER,
            password=DB_PASSWORD
        )

    def get_connection(self):
        return self.connection_pool.getconn()

    def return_connection(self, connection):
        self.connection_pool.putconn(connection)

# Redis caching for frequent queries

    cache_key = f"user:{user_id}"

        return json.loads(cached_user)
